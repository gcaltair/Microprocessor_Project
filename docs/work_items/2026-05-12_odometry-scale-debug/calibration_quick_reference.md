# 里程计标定速查表

## 适用场景

用于新版固件下的单段量距标定。推荐配合以下流程使用：

1. 发送 `R0`
2. 发送 `P1.0,0.0` 或 `P-1.0,0.0`
3. 等待停车后发送 `O`
4. 测量实体实际距离
5. 发送 `D实际距离`

---

## 先看什么

优先看两类输出：

### 1. `O` 里的 `LMOVE`

示例：

```text
LMOVE valid=1 cmd=1.000 progress=0.957 dl=0.793 dr=1.176
```

字段含义：

- `valid=1`：最近一次相对位移段快照有效
- `cmd`：这一段命令距离
- `progress`：控制器认定的这一段进度
- `dl / dr`：这一段左轮/右轮单段轮程

判读规则：

- 如果 `valid=0`，这一轮不要用于标定，先重做单段测试
- 如果 `cmd` 和你刚测的方向不一致，说明这一轮数据不是你想看的那一段
- `dl / dr` 比实体距离都大，说明该方向轮程记大了
- `dl / dr` 一大一小差很多，说明左右轮不对称明显

### 2. `D...` 里的 `CAL diag`

示例：

```text
CAL diag left_err=-3.3% right_err=43.4% asym=39.0%
```

字段含义：

- `left_err`：左轮相对实体距离的误差百分比
- `right_err`：右轮相对实体距离的误差百分比
- `asym`：左右轮不对称程度

判读规则：

- `err > 0`：该轮里程记大了，系数应调小
- `err < 0`：该轮里程记小了，系数应调大
- `asym` 越大，左右轮越不对称

经验上可先这样看：

- `|err| < 5%`：该轮基本可接受
- `5% ~ 10%`：建议继续调
- `> 10%`：该轮明显有偏差
- `asym > 10%`：左右轮差异已经值得优先处理

---

## `Klf/lr/rf/rr` 到底对应什么

- `lf`：左轮前进系数
- `lr`：左轮倒车系数
- `rf`：右轮前进系数
- `rr`：右轮倒车系数

所以：

- 前进测试 `P1.0,0.0` 主要更新 `lf` 和 `rf`
- 倒车测试 `P-1.0,0.0` 主要更新 `lr` 和 `rr`

不要把前进测试结果直接拿去改倒车系数，反过来也一样。

---

## 直接怎么用 `D...` 的建议

如果串口打印：

```text
Use: K1.034,1.000,0.697,1.000
```

意思是：

- 建议把 `lf` 改成 `1.034`
- `lr` 先不动
- 建议把 `rf` 改成 `0.697`
- `rr` 先不动

也就是直接发送：

```text
K1.034,1.000,0.697,1.000
```

如果你正在做倒车测试，固件会反过来只改 `lr / rr`，保持 `lf / rf` 不变。

---

## 怎么判断这轮结果更像什么问题

### 情况 A：`MOVE progress` 接近目标，但实体明显没走够

例如：

- `cmd=1.000`
- `progress=0.95`
- 实体只走 `0.82 m`

更像：

- 轮程比例偏差
- 左右轮不对称

不太像：

- 相对位移进度统计本身少算

### 情况 B：左右轮 `err` 都为正，而且都偏大

更像：

- 该方向整体轮程记大了
- 两边都要往下调

### 情况 C：一边 `err` 很大，另一边接近 0

更像：

- 单侧轮比例偏差更严重
- 优先改单侧对应的 `K`

### 情况 D：`asym` 很大

更像：

- 左右轮不对称是主要矛盾
- 先解决左右差异，再追整体精度

### 情况 E：`LMOVE valid=0` 或 `CAL source=cumulative`

说明：

- 当前建议不是基于干净的单段运动
- 这一轮不要急着回写系数

应对：

1. `R0`
2. 只做一段 `P...`
3. 立刻 `O`
4. 立刻 `D...`

## 结合你当前首轮前进数据的例子

已知：

- 实体距离：`0.82 m`
- `LMOVE` 近似参考：左轮 `0.793 m`，右轮 `1.176 m`

则：

- 左轮略小，误差接近可接受边缘
- 右轮明显记大
- 左右不对称很明显

所以当前前进方向更像：

- `rf` 需要明显下调
- `lf` 只需要小幅上调或微调

这和首轮 `D0.82` 给出的方向是一致的：

```text
Use: K1.034,1.000,0.697,1.000
```

---

## 推荐的复测节奏

### 前进

1. `R0`
2. `P1.0,0.0`
3. `O`
4. 量距离
5. `D实际距离`
6. 如果建议合理，应用新的 `Klf,lr,rf,rr`
7. 再重复一轮前进测试

### 倒车

1. `R0`
2. `P-1.0,0.0`
3. `O`
4. 量距离
5. `D-实际距离`
6. 如果建议合理，应用新的 `Klf,lr,rf,rr`
7. 再重复一轮倒车测试

---

## 本轮人工回传时最有用的内容

直接把下面几行发回来就够：

```text
O 里的 LMOVE:
D... 里的 CAL source:
D... 里的 CAL diag:
D... 里的 Use: K...
实体实际距离:
```

有这几行，我就能更快判断下一轮该不该直接回写系数，还是先继续排查控制或转向问题。
