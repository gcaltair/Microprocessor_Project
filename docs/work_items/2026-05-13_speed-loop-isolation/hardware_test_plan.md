# 速度环隔离测试计划与记录

## 测试元数据

- 日期：2026-05-13
- 作者：AI agent
- 功能：速度环隔离测试命令 `WSleft,right`
- 串口：`COM15`，`961200` baud
- 测试状态：已在小车悬空状态下执行

## 目标

确认在绕过位置环和角度环后，左右轮速度 PID 是否仍会导致低速抖动、PWM 大幅摆动或左右轮响应明显不一致。

## 前置条件

- 小车悬空，允许正反向和单轮测试。
- 操作者允许 agent 通过串口发送测试命令。
- 每个测试段都使用 `S` 停止。

## 已执行命令类型

- 默认参数基线：`PKL/PKR = 8000,33705,33`
- P-only sweep：`Kp = 2000 / 2500 / 3000 / 4000`，`Ki = 0`，`Kd = 0`
- 小积分 sweep：`Kp = 2500`，`Ki = 300 / 800 / 1500`，`Kd = 0`
- 对照组：`Kp = 3000`，`Ki = 500`，`Kd = 0`
- 最终候选长测：`Kp = 2500`，`Ki = 800 / 1500`，`Kd = 0`
- 最终确认：`PKL,2500,800,0` 与 `PKR,2500,800,0`

典型命令序列：

```text
S
PKL,2500,800,0
PKR,2500,800,0
R0
WS0.06,0.06
O
...
S
WS-0.06,-0.06
O
...
S
```

## 关键观察

默认速度环参数：

- `Kp=8000 Ki=33705 Kd=33`
- 在悬空 `WS0.04,0.04` 和 `WS-0.04,-0.04` 下出现明显振荡。
- 实际轮速曾出现约 `0.28~0.46 m/s` 的尖峰，远高于 `0.04 m/s` 目标。
- PWM 峰值达到数千，并出现正负来回翻转。

P-only 参数：

- `Kp=2000~4000 Ki=0 Kd=0` 均显著降低振荡。
- PWM 峰值降到几百以内。
- `0.06 m/s` 附近比 `0.04 m/s` 更可测，因为低速编码器量化导致 `0.04 m/s` 经常显示为 `0` 或一个离散台阶。

积分参数：

- `Ki=300 / 800 / 1500` 都远比默认 `33705` 安全。
- `Kp=3000 Ki=500` 曾出现右轮 `0.379 m/s` 尖峰，不作为默认候选。
- `Kp=2500 Ki=1500` 在换向和较高速度段更容易出现符号异常。
- `Kp=2500 Ki=800 Kd=0` 在双轮、单轮、正反向测试中综合最稳。

最终确认样本：

- `WS0.06,0.06`：左右轮多数采样约 `0.045~0.097 m/s`，PWM 多数低于 `150`。
- `WS-0.06,-0.06`：左右轮多数采样约 `-0.010~-0.054 m/s`，PWM 多数低于 `150`。
- `WS0.00,0.06`：左轮保持 `0.000`，右轮约 `0.050~0.054 m/s`，单侧 PWM 有一次约 `241`。
- `WS0.00,-0.06`：左轮保持 `0.000`，右轮约 `-0.045~-0.080 m/s`，PWM 峰值约 `125`。

## 结论

速度环原默认参数过激，是低速悬空测试中 PWM 大幅摆动和轮速尖峰的重要来源。当前推荐默认值：

```text
PKL,2500,800,0
PKR,2500,800,0
```

该参数已写入固件默认 `PID_system_init()`，重启后会自动生效。

## 后续地面验证

悬空测试不能完全代表地面负载。刷入新固件后建议继续做：

1. 小车落地，空旷地面。
2. `S`
3. `R0`
4. `WS-0.04,-0.04`
5. 连续发送 `O`，观察 3 到 5 秒。
6. `S`
7. 若稳定，再做 `P-0.30,0` 和 `P-1.00,0`。

观察重点：

- 低速倒车是否仍左右摆动。
- `ODOM ls/rs` 是否频繁反号。
- `CTRLDBG pwm` 是否重新出现数千级尖峰。
- 地面负载下是否需要略提高 `Ki` 或加入前馈/死区补偿。

## 2026-05-14 诊断补充

地面继续调参时出现两个新现象：

- 前进加速到目标速度较慢。
- 原地转向仍不够快。

本轮不再继续盲目增大 PID 参数，而是新增诊断输出：

- `PIDDBG L/R/A/P`：左右速度环、角度环、位置环的 setpoint/current/error/P/I/D/output/integral/sat。
- `LIMDBG`：位置环 base speed 原始值与限幅值、终端限速、角度 turn 输出限幅、角度死区激活状态。
- `MOTORDBG`：PID PWM、加 `MOTOR_DEAD_ZONE` 后实际写给电机的 PWM、左右电机方向。
- `ENCDBG`：每个控制周期的编码器脉冲增量、原始轮速、滤波轮速、odometry 缩放轮速。

安全限时测试命令本轮未实现；后续测试仍使用 `S` 停车和短时手动采样。

## 2026-05-14 第一轮参数改动

重新烧录诊断固件后，使用 `PKL,1800,3000,0` 和 `PKR,1800,3000,0` 做短测：

- `WS0.10,0.10`：最大滤波轮速约 `0.224 m/s`，最大 PID PWM 约 `559`。
- `WS0.12,0.12`：最大滤波轮速约 `0.388 m/s`，最大 PID PWM 约 `765`。
- `P0.20,0`：完成到 `progress=0.150`、`remain=0.000`，最大 PID PWM 约 `416`，期间 `term_lim` 多次触发。
- `A90`：约 `3.5 s` 只转到 `70.8 deg`，`PIDDBG A sat` 和 `LIMDBG tsat` 基本持续触发。
- `A-90`：约 `3.5 s` 转到 `-90.4 deg`，但也存在角度环限幅。

结论：

- 地面候选 `Kp=1800 Ki=3000 Kd=0` 比默认 `2500,800,0` 更适合当前地面前进和转向。
- 转向慢的直接软件原因之一是原地角度输出被 `TURN_OUTPUT_LIMIT_INPLACE=0.08 m/s` 限死。
- 已将默认速度环改为 `1800,3000,0`。
- 已将 `TURN_OUTPUT_LIMIT_INPLACE` 从 `0.08 m/s` 提高到 `0.12 m/s`。
- 已修正手动/转向模式下 `LIMDBG` 位置限幅字段残留的问题。
