# 终点与转向抖动硬件测试计划

## 测试元数据

- 日期：2026-05-13
- 作者：Codex
- 功能：相对位移终点减速、移动中角度修正限幅、转向死区迟滞
- 固件构建：待填写
- 硬件版本：当前 STM32F446 小车

## 目标

确认新版控制逻辑是否降低以下问题：

- `P1,0` 快到达终点时明显抖动、摆头或左右轮反向拉扯。
- `A90 / A-90` 或 `L / R` 转角时在目标角附近反复抖动。
- `O` 输出的 `CTRLDBG` 能帮助判断是否仍存在角度修正过强。

## 前置条件

- 小车电池电量充足。
- 编码器线束已重新整理并固定。
- 当前默认编码器系数应为 `K0.957,0.992,0.896,1.030`。
- 测试地面平整，前方至少保留 `1.5 m` 安全距离。
- 操作者随时准备发送 `S` 停车。

## 命令序列

### 测试 A：直线终点抖动

1. `R0`
2. `K`
3. `O`
4. `P1,0`
5. 动作结束后立即发送 `O`
6. 实测实际前进距离，发送 `D<实际距离>`，例如 `D0.96`
7. 重复步骤 1-6 共 `3` 轮

### 测试 B：反向终点抖动

1. `R0`
2. `P-1,0`
3. 动作结束后立即发送 `O`
4. 实测实际后退距离，发送 `D<实际距离>`，例如 `D-0.96`
5. 重复 `3` 轮

### 测试 C：原地转向抖动

1. `R0`
2. `A90`
3. 动作稳定后发送 `O`
4. `A-90`
5. 动作稳定后发送 `O`
6. 如需更小角度测试，重复 `A10`、`A-10`

## 预期行为

- `P1,0` 接近终点时应逐渐减速，最后 `15 cm` 内速度应明显低于中段速度，不应出现明显一侧轮子反向拉扯。
- `CTRLDBG turn` 在移动终点附近应随 `base` 限制，不应远大于基准速度。
- `CTRLDBG l_sp/r_sp` 在直线前进终点附近不应频繁一正一负。
- 动作结束后 `O` 中的 `LCTRL` 应保留最后一轮驱动控制输出；若现场无法在终点前实时发送 `O`，优先记录 `LCTRL`。
- 原地转向后 `O` 中的 `TCTRL` 应保留最近一次转向控制输出；若现场无法在转角抖动时实时发送 `O`，优先记录 `TCTRL`。
- `A90 / A-90` 在目标角附近应减少反复启停，最终能进入稳定静止。
- 最终位移误差可先接受在 `5-10 cm` 范围内，本轮重点是抖动是否下降。

## 失败信号

- 终点前 `l_sp/r_sp` 频繁一正一负。
- `LCTRL l_sp/r_sp` 一正一负，说明终点前最后一轮仍发生左右轮反向拉扯。
- `TCTRL turn` 较大或 `pwm` 较大但 `ang_err` 已很小，说明转角目标附近仍在过度修正。
- `CTRLDBG ang_err` 在小角度范围内来回跳，但 `turn` 仍反复输出较大值。
- PWM 正负频繁翻转。
- 小车在目标点附近持续抖动超过 `2 s`。
- 转向接近目标角后仍持续左右摆动。

## 安全停止 / 恢复

- 停车命令：`S`
- 状态复位：`R0`
- 如出现持续抖动或冲出测试区域，先发送 `S`，再断电检查线束和轮胎状态。
