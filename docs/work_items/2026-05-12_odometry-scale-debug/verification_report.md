- 基线结果：`R0` 后 `ODOM / MOVE / EST / CTRL` 均回到零附近，说明复位链路基本干净
- 手动转向结果：`L / R` 已恢复为相对转向语义，但转向过程仍有卡顿和可能的超调
- 前进 1.0 m 结果：`ODOM x=0.984 m`、`MOVE progress=0.957 m`，实体仅约 `0.82 m`，`ENC dl/dr = 0.793 / 1.176 m`
- 倒车 1.0 m 结果：记录里的 `ENC dl/dr = -0.337 / 0.430 m` 是累计量，不是独立倒车段。若与前进阶段累计值做差，倒车单段更接近 `dl/dr = -1.130 / -0.746 m`，说明问题更像反向左右轮不对称和比例偏差，而不能直接下结论说“符号链路错误”
- 过程限制：本轮记录在前进和倒车前没有重新 `R0`，且 `D0.82` 建议基于累计轮程，混入了前序摆位和前一段运动，不能直接当作单段标定真值
- 已补充固件改进：`D...` 现优先使用最近一次完成的 `P{x},{y}` 相对位移段轮程，避免累计残留污染标定建议；`O` 也新增 `LMOVE ...` 直接显示最近一次单段轮程；`D...` 还新增 `CAL diag ...` 直接打印左右轮误差百分比和不对称程度
- 第二轮前进结果：实体 `0.83 m`，`LMOVE dl/dr = 0.838 / 1.069 m`，`left_err=1.0%`、`right_err=28.8%`、`asym=24.2%`
- 第二轮倒车结果：实体 `-1.12 m`，`LMOVE dl/dr = -1.048 / -0.857 m`，`left_err=-6.4%`、`right_err=-23.5%`、`asym=20.1%`
- 当前合并系数建议：`K0.990,1.069,0.776,1.308`
- 当前判断：前进方向主要是右轮前进系数过大；倒车方向两侧系数都偏小，其中右轮倒车偏差更大；`MOVE progress` 不是本轮主误差源
- 第三轮复测确认：`K0.990,1.069,0.776,1.308` 已生效，`O` 显示 `cal=(0.990,1.069,0.776,1.308)`
- 第三轮前进结果：实体 `0.97 m`，`LMOVE dl/dr = 0.882 / 1.020 m`，`left_err=-9.0%`、`right_err=5.2%`、`asym=14.5%`
- 第三轮倒车结果：实体 `-0.85 m`，`LMOVE dl/dr = -0.864 / -1.040 m`，`left_err=1.6%`、`right_err=22.4%`、`asym=18.5%`

## 仍然存在的缺口
